## 跟你朋友介紹 Git
如何使用 Git 來管理笑話。菜哥的笑話，會演進常常有版本一、版本二......版本十。

### 什麼是 Git ?
Git 是一個版本控制的系統。

### 什麼是版本控制？
最簡單的版本控制就是保留不同的版本檔案。如果更好一點的是用固定格式來編號管理。
例如說：
joke-2019-01-01 第一版
joke-2019-01-02 第一版
joke-2019-01-03 第一版
joke-2019-01-03 第二版
這樣做的好處是，什麼時間的第幾個版本很清楚。
缺點就是
1. 不清楚他們之間的差異。
2. 檔案多了之後會很難管理。

所以有了 Git 這樣的版本控制系統來協助管理。

### 版本控制的原理
今天如果蔡哥要用 Git 來管理笑話，用Git 的好處是一次只會顯示一個版本的笑話。
所以就可以建立一個資料夾，例如資料夾名稱：joke 
然後把當前版本的笑話放進 joke 資料夾，接著設定好 Git 管理。
Git 管理的模式就有點像利用資料夾管理，每次提交之後，就會產生一個新資料夾，並且儲存起來，這個是我們平常看不到的。接著又可以再次修改內容，然後提交，又會額外建立新資料夾，所以每個版本都可以儲存起來，最棒的是 Git 還會幫我們管理先後順序，這樣就可以透過 Git 管理笑話了。

### 如何操作 Git
假設蔡哥已經學會 CLI 的基本操作了。
所以 Git 可以透過 Cmder 來控制：
#### 初始化 Git & 建立第一個版本：
1. 先切換到 joke 的資料夾，例如輸入 `cd D:\cmder\joke`，如果蔡哥的電腦在不同的硬碟，例如說在 E 槽那就先輸入 `E:` 就可以切換到 E 槽，接著在輸入 `cd E:\joke`，就可以正確切換了
2. 接著輸入 `git init` 初始化並建立 Git repository。
3. 等待建立完成之後，輸入 `git status` 可以檢查現在的狀況如何，可以看到 `Untracked files:` 沒有追蹤的檔案，這邊會顯示還沒被管理的檔案。
4. 所以我們要把那些檔案加入版本控制，這時候可以輸入`git add 檔案名稱` 加入控制，例如：`git add joke1.txt`、`git add joke2.txt` 或是想要全部加入的話，直接輸入 `git add .` 就可以把該資料夾底下的東西全部加入管理。
5. 再次輸入 `git status` 可以看到寫 No commits yet 代表還沒提交。
6. 如果要提交，就輸入 `git commmit -m "版本訊息"`，例如輸入`git commit -m "第一版"` ，系統就會把這個已經添加的版本給提交上去版本訊息是"第一版"，這時候就可以輸入 `git log`，就會顯示 commit + 一串亂碼，像是`commit 51da2ffbde2af027f047089be*******9b1411d3` ，然後下面會寫出此版本的訊息，這個就像是，Git 幫我們創建一個新資料夾 名字叫作`51da2ffbde2af027f047089be*******9b1411d3` 並幫我們另外儲存。 
7. 這樣就完成了，第一個版本了。之後可以修改了，修改完成之後記得 add 之後在 commit，那就會變成第二版。

#### 修改並提交成新版本
假設今天改了 joke1.txt 的內容，想要把他提交。
1. 可以輸入 `git diff joke1.txt` 當前修改（未 add）跟最新 commit 的差異。也可以確認修改有無錯誤，當然這步驟是可以忽略的。
2. 接著輸入 `git add joke1.txt` 加入控制，再輸入 `git commit -m "第二版"`。就可以成功 commit 了。也可以直接輸入 `git commit -am "第二版"` 同時加入控制且 commit。這時候就類似系統就會額外建立一個新資料夾，並且把檔案都儲存進去。
#### 我想要看之前的版本 
1.  輸入指令 `git log --oneline`  。加上`--oneline` 這是會使 `git log` 簡短顯示的版本，通常是因為切換的時候其實不用輸入這麼多字就可以切換。
2. 這時候可以從指令頁面上看到每個版本的簡碼，假如我要切到其中一個版本，該版本簡碼是 8c6fda7 這時候只要輸入`git checkout 8c6fda7` ，就會把版本切到那個版本了。
3. 這時候就可以看看這個版本的笑話長什麼樣子。
4. 在看完之後切記得切回最新版本，輸入`git checkout master` 切到最新版本。

### Git 指令的複習
`git init` 初始化並建立 repository 
`git status` 查看當前的狀態
`git log` 查看各個 commit 的版本
`git log --oneline` 顯示簡短的 log
`git add` 添加到版本控制，如果有新檔案務必要用這個指令才能添加進版本控制
`git commit -m "版本說明"`  提交
`git checkout "版本編碼"` 切換版本
